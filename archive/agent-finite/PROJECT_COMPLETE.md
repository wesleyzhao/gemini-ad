## Deployment Instructions

### 1. Configure Google Analytics (Required)
```bash
# 1. Create GA4 property in Google Analytics
# 2. Get Measurement ID (G-XXXXXXXXXX)
# 3. Update scripts/add-analytics.js with real ID
# 4. Run update script
npm run analytics:add
```

### 2. Deploy to GitHub Pages
```bash
# 1. Push to main branch
git push origin main

# 2. Enable GitHub Pages in repository settings
# Settings > Pages > Source: main branch

# 3. Wait for deployment (~2 minutes)

# 4. Access at: https://[username].github.io/[repo-name]/
```

### 3. Verify Deployment
- [ ] All pages load correctly
- [ ] Analytics tracking works
- [ ] Responsive design on mobile/tablet/desktop
- [ ] No console errors
- [ ] No broken links

### 4. Custom Domain (Optional)
```bash
# 1. Add CNAME file with your domain
echo "your-domain.com" > CNAME

# 2. Configure DNS settings with your registrar
# 3. Enable HTTPS in GitHub Pages settings
```

## Known Limitations

### 1. Google Analytics Placeholder
- **Issue:** GA4 ID is placeholder (G-XXXXXXXXXX)
- **Impact:** Analytics won't track until configured
- **Fix:** Update ID in `scripts/add-analytics.js` and run `npm run analytics:add`

### 2. Tests Require Local Server
- **Issue:** Playwright tests need `npm run dev` running
- **Impact:** Tests fail without server
- **Fix:** Always run `npm run dev` before testing

### 3. Some Inline Styles
- **Pages:** email-savior.html, workflow-wizard.html, writers-room.html
- **Impact:** Inline styles override shared design system
- **Note:** This is intentional and doesn't affect functionality
